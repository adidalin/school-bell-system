## 议题1：区域是一等公民

**张老师**：当前系统最大的问题是"区域"只是个附属属性。实际上区域应该是一等公民——课表属于区域、今日状态属于区域、明日状态属于区域。用户说"课表按区域执行"，意思是每个区域有自己独立的课表方案。

**陈工**：数据模型要改。当前结构是 `schedule → tasks(zone_id)`，应该是 `zone → schedule → tasks`。每个区域绑定自己的课表方案。

**王工**：对，教学楼周一到周五按"平日课表"走，操场可能只有上午用"课间操课表"，宿舍可能有独立的"作息课表"。它们是完全独立的。

**结论**：
- 每个区域绑定一个独立的课表方案（schedule_id）
- 首页按区域显示今日状态、明日状态
- 今日/明日控制都是按区域操作
- 保留全局快捷操作（全局暂停/全局恢复/全局打铃）

## 议题2：今日/明日状态控制

**张老师**：用户的核心场景是这样的——
- 今天是周六（休息日），系统显示"今日不打铃"。但校长说今天要补课，老师点击"教学楼-今日打铃"，选择"平日课表"，教学楼今天就会打铃。
- 明天是周日，但明天要补周一的课。老师提前设置"教学楼-明日打铃，使用周一课表"。
- 今天是工作日，但教学楼装修停课。老师设置"教学楼-今日不打铃"。

**王工**：所以每个区域每天有三种状态：
1. **自动**（默认）— 系统根据工作日/休息日自动判断
2. **强制打铃** — 即使休息日也打铃，需选择使用哪个课表
3. **强制不打铃** — 即使工作日也不打铃

**陈工**：明日状态同理。提前一天设置好明天的状态，第二天自动生效。

**结论**：
- 今日控制：每个区域独立设置（自动/今日打铃/今日不打铃）
- 明日控制：每个区域独立设置（自动/明日打铃/明日不打铃）
- 打铃时可选择使用哪个课表方案
- 首页醒目显示各区域今日/明日状态

## 议题3：钉钉机器人告警逻辑

**张老师**：用户明确说了告警逻辑——
1. 周一到周五应该打铃，周六周日应该不打铃
2. 如果不是这种正常状态（比如周五不打铃、周日打铃），就提前一天早上8点提醒
3. 关键设置变更也要提醒
4. 异常信息也要提醒

**王工**：具体来说，钉钉告警分三类：

**第一类：明日异常状态提醒（每天早上8:00）**
- 如果明天某个区域的状态不是"常规状态"（工作日打铃、休息日不打铃），就发提醒
- 举例：明天周日但教学楼要打铃 → 提醒"明天(周日)教学楼将执行打铃，使用平日课表"
- 举例：明天周五但宿舍不打铃 → 提醒"明天(周五)宿舍将不执行打铃"
- 这让管理员提前确认，避免误设

**第二类：实时异常告警（即时）**
- 铃声文件缺失（播放前检测到文件不存在）
- 打铃失败（播放过程中出错）
- 音频设备离线/异常
- NTP校时失败
- 30分钟内同类告警不重复

**第三类：关键操作提醒（即时）**
- 有人修改了课表
- 有人修改了区域设置（音量、设备等）
- 有人修改了系统设置（NTP、钉钉等）

**陈工**：同一条告警30分钟内不重复发送，避免刷屏。

**结论**：
- 钉钉告警三类：明日异常预提醒(8:00) + 实时异常告警 + 关键操作提醒
- 明日预提醒：检测明天各区域是否偏离常规状态
- 实时异常：文件缺失、播放失败、设备异常、NTP失败
- 关键操作：课表变更、设置变更
- 30分钟去重

## 议题4：NTP校时显示

**王工**：用户说"互联网时间为什么没显示"。当前NTP只在后台静默执行，界面上看不到。应该在首页或设置页醒目显示：
- 当前NTP状态（已同步/未同步）
- 上次校时时间
- 时间偏差
- 下次校时时间

**张老师**：建议在首页顶部的状态栏显示一个小时钟图标，鼠标悬停或点击可以看到NTP详情。

**结论**：
- 首页header显示NTP同步状态（小图标+文字）
- 点击可展开查看详情（上次校时、偏差、服务器）

## 议题5：额外功能优化（调研结果汇总）

**张老师**：根据市场调研和同类产品分析，我们还需要补充以下功能：

**5.1 课表按星期筛选**
- 每个区域的课表可以按星期几筛选任务
- 比如教学楼周一有升旗仪式、周五提前放学
- 当前已有weekday字段，但前端没有按星期筛选展示

**5.2 日程预览**
- 可以查看未来7天的打铃计划
- 按区域分别展示
- 标注节假日、调休、临时调整

**5.3 铃声文件自检**
- 系统启动时自动检查所有铃声文件是否存在、是否损坏
- 缺失/损坏的文件在界面标红提醒
- 播放前再次检查文件可用性

**5.4 操作日志完善**
- 记录所有关键操作：修改课表、修改设置、手动打铃、今日/明日状态变更
- 支持按类别筛选
- 显示操作人（如果有多用户）

**5.5 课表导入优化**
- 支持Excel模板导入（按区域分sheet）
- 导入前预览确认
- 导入后显示变更摘要

**5.6 系统健康看板**
- CPU/内存使用率
- 音频设备状态
- 磁盘空间
- 服务运行时长

## 执行计划

### 第一步：数据模型重构
- zones表增加 schedule_id 字段（绑定默认课表）
- today_exceptions 表增加 schedule_id 字段（已有）
- 新增 tomorrow_overrides 表（明日状态覆盖）

### 第二步：引擎逻辑重构
- check_and_ring() 改为按区域独立判断
- 每个区域查自己的课表、自己的今日/明日状态
- 钉钉告警：新增每日8:00明日状态检查任务
- 铃声播放前自检文件可用性

### 第三步：API重构
- /api/status 增加明日状态、NTP详情
- /api/today_control 改为按区域操作
- 新增 /api/tomorrow_control
- 新增 /api/ntp_status
- 铃声文件自检接口

- 首页：今日/明日状态按区域显示
- 课表页：按区域切换显示

- 钉钉配置：完整中文说明
