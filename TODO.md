# 后期开发计划 & 待解决问题

## 一、PocketBase 权限系统集成

### 目标
Flask 接入 PocketBase 做鉴权，实现角色级数据隔离。

### 角色与权限

| 角色 | 可见范围 | 可操作 |
|------|---------|--------|
| `admin` | 全部区域 | 全部权限 |
| `primary`（小学） | 仅小学区域 | 编辑课表、上传铃声、查看日志 |
| `middle`（初中） | 仅初中区域 | 编辑课表、上传铃声、查看日志 |
| `uploader` | 无区域 | 仅上传铃声 |

### 实现要点

- PocketBase 与 Flask 同机运行，端口 `8090`
- Flask 加 JWT 中间件，验证每个请求的令牌
- 用户表扩展 `zone_id` 和 `role` 字段
- 前端加登录页，路由守卫
- `run.py` 启动时自动拉起 PocketBase 子进程

### 不改的部分

- 不打铃调度（调度逻辑不变）
- 不迁移数据（PocketBase 仅存用户，区域/课表/铃声仍在 SQLite）
- 不改造音频引擎

### 状态

> ⬜ 未开始

## 三、音频口漂移检测

### 现象
WDM-KS 流可能因驱动/硬件原因静默死亡，虽然已加 `st.active` 健康检测重建，但无法100%覆盖所有场景。

### 待实现

- [ ] **日志告警**：流重建时写告警日志，便于排查是否频繁漂移
- [ ] **流健康看板**：前端页面显示当前所有音频流的状态（创建时间、是否活跃）
- [ ] **每日设备快照**：启动时记录可用设备列表，对比异常变化
- [ ] **静音检测**：播放时检测回调输出是否持续为零（无声故障），超过阈值则重建

### 状态

> ✅ 流重建已实现（需后续增强监控和告警）

- 新版"排课管理"与旧版"课表管理"功能重叠，后期合并为一套

### 4.4 多语言/编码兼容

### 4.5 数据库备份与恢复
- 每日自动备份 `data/school_bell.db`
- 保留最近7天备份

### 4.6 音频设备枚举稳定性
- WDM-KS 设备索引可能随 PortAudio 初始化顺序变化，当前用 `name:` 格式定位，但如果设备名发生变化（驱动更新），会导致查找失败
- 考虑加入设备名模糊匹配或手动指定 host API
